Plants VS Zombies

Little Game with WPF

Project Description

The goal of this project was to make a little 2D game with WPF and a basic game engine by the IUT de Dijon.
As this framework was primary, I first made a big update on it. The base engine can only show images and move them to an absolute position. My update added the option to move them from their relative position, to give them a collision (rectangle, ellipse, polygon, point, line), multiple animations (position, rotation, scale, opacity, pattern texture) and a layer because otherwise the layer was just the order of drawing, which is just dumb. I also added multi-stream musics with the option to loop, to add a loop point, to switch between channels and musics with a transition and to play sound effects. I also changed how the game timer works: teachers based their timer on a waiting system but the problem was that it doesn't support the fact the code never run at a smooth frequence so I changed it to be relative to the real time. Then I added features to the game engine, like the option to search for a sprite of a specific type, which we needed in some rare cases.
Then, with my team, we planned on making a Plants VS Zombies game, even if it was 2.5D instead 2D, because we had the layer system. From there, I made all the engine and scene stuff, and a big part of the UI. For the sprites, we used the textures from the original game and made the pattern texture animations. The rest of my team made the level system, and most of the game logic.
